3.4
USING PARSER GENERATORS
The task of constructing LR(1) or LALR(1) parsing tables is simple enough
to be automated. And it is so tedious to do by hand that LR parsing for real-
istic grammars is rarely done except using parser-generator tools. Yacc (“Yet
another compiler-compiler”) is a classic and widely used parser generator;
Bison
and
occs
are more recent implementations.
A Yacc speciﬁcation is divided into three sections, separated by
%%
marks:
parser declarations
%%
grammar rules
%%
programs
The
parser declarations
include a list of the terminal symbols, nonterminals,
and so on. The
programs
are ordinary C code usable from the semantic ac-
tions embedded in the earlier sections.
The
grammar rules
are productions of the form
exp :
exp PLUS exp
{
semantic action
}
where
exp
is a nonterminal producing a right-hand side of
exp+exp
, and
PLUS
is a terminal symbol (token). The
semantic action
is written in ordinary
C and will be executed whenever the parser reduces using this rule.
